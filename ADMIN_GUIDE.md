# 📋 راهنمای مدیریت سیستم رَدیف

## 🔐 اطلاعات ورود ادمین

```
Identifier: admin@radif.local
Password:   Admin@1234
```

**⚠️ مهم:** بعد از اولین ورود، حتماً رمز عبور را تغییر دهید!

---

## 🚀 نحوه استفاده

### 1️⃣ **ورود به پنل ادمین**

1. به آدرس `http://localhost:5173/auth` بروید
2. روی تب **ورود** کلیک کنید
3. اطلاعات ادمین را وارد کنید
4. بعد از ورود، به صورت خودکار به داشبورد هدایت می‌شوید

### 2️⃣ **دسترسی به داشبورد**

مستقیماً به آدرس `http://localhost:5173/dashboard` بروید.

---

## 📊 امکانات داشبورد ادمین

### ✅ **داشبورد اصلی (Overview)**
- نمایش آمار کلی (تعداد نوبت‌ها، مقالات، دانلودها)
- لاگ آخرین فعالیت‌ها

### ✅ **مدیریت نوبت‌ها (Appointments)**
- مشاهده تمام نوبت‌های رزرو شده
- تغییر وضعیت نوبت (Pending → In Progress → Completed)
- حذف نوبت
- مشاهده اطلاعات کامل (نام، شماره، خودرو، ایسیو، کد پیگیری)

### ✅ **پیام‌های تماس (Contact Messages)**
- مشاهده تمام پیام‌های ارسال شده از فرم تماس
- تغییر وضعیت پیام (New → Read → Replied)
- اطلاعات کامل (نام، شماره، موضوع، متن پیام)

### ✅ **مدیریت دانشنامه (Wiki Manager)**
- آپلود مقالات و فایل‌های دامپ جدید
- دسته‌بندی محتوا (ایسیوها، مالتی‌پلکس، کدهای خطا، فایل‌های دامپ)
- تنظیم دسترسی (رایگان / نیاز به ورود)
- انتشار / پیش‌نویس مقالات
- حذف مقالات
- مشاهده تعداد دانلودها

---

## 👤 سیستم کاربری

### **User (کاربر عادی)**
- ثبت‌نام و ورود
- رزرو نوبت با Auto-fill اطلاعات پروفایل
- دسترسی به مقالات بر اساس تنظیمات

### **Admin (ادمین)**
- تمام امکانات کاربر عادی
- دسترسی کامل به داشبورد مدیریت
- مدیریت نوبت‌ها، پیام‌ها و محتوای دانشنامه

---

## 🔧 ساخت ادمین جدید

برای ساخت یوزر ادمین جدید:

```bash
cd backend
node scripts/createAdmin.js
```

یا به صورت دستی در MongoDB:

```javascript
db.users.updateOne(
  { identifier: "email@example.com" },
  { $set: { role: "admin" } }
)
```

## 📝 نکات مهم

### ✅ **Auto-fill فرم Booking**
وقتی کاربر لاگین کرده باشد و به صفحه `/booking` برود:
- نام از پروفایل کاربر خوانده می‌شود
- شماره تماس از فیلد `phone` یا `identifier` خوانده می‌شود

### ✅ **محافظت از روت‌های ادمین**
- همه روت‌های مدیریتی نیاز به احراز هویت + نقش ادمین دارند
- در صورت دسترسی غیرمجاز، خطای `403 Forbidden` برگردانده می‌شود

### ✅ **CORS تنظیم شده**
- در محیط development: همه originها مجاز هستند
- در محیط production: فقط `CLIENT_URL` از `.env` مجاز است

## 🐛 عیب‌یابی

### مشکل: نمی‌توانم وارد داشبورد شوم
✅ مطمئن شوید که:
1. با اکانت ادمین وارد شده‌اید (چک کنید `user.role === 'admin'`)
2. توکن منقضی نشده باشد (30 روز اعتبار دارد)
3. کوکی‌ها در مرورگر فعال باشند

### مشکل: Auto-fill کار نمی‌کند
✅ مطمئن شوید که:
1. کاربر لاگین کرده باشد
2. فیلدهای `name` و `phone` در پروفایل کاربر پر شده باشند

### مشکل: CORS Error
✅ چک کنید:
1. `NODE_ENV=development` در `.env` فایل backend
2. سرور backend را restart کنید
